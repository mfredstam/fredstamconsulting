# Test Suite Design — Fredstam Consulting Static Site

**Date:** 2026-03-14
**Status:** Approved

## Overview

Add an automated test suite to the static site that runs on every PR. The site itself remains static (no build step, no server runtime). Test tooling is dev-only infrastructure managed via `package.json`.

## Goals

- Catch regressions in user-interaction behaviour (hamburger, nav, footer) before they reach `main`
- Enforce accessibility (WCAG 2.1 AA) on every PR via axe-core
- Validate HTML structure and aria usage via html-validate

## Non-goals

- Unit testing JS modules in isolation (no jsdom, no mocking)
- Visual regression / screenshot diffing
- Testing against the live GitHub Pages URL

## Tools

| Tool | Purpose | Version |
|---|---|---|
| `@playwright/test` | Browser automation + test runner | latest |
| `@axe-core/playwright` | Accessibility audit within Playwright | latest |
| `html-validate` | HTML structure validation (no browser) | latest |
| `serve` | Static file server used by Playwright `webServer` | `^14` |

Chromium is the only browser target. Firefox and WebKit are out of scope.

## File Structure

```
package.json                    ← devDependencies + test scripts
package-lock.json               ← generated by npm install, committed to git
playwright.config.js            ← Playwright config (Chromium, webServer, baseURL)
.htmlvalidate.json              ← html-validate rules config
tests/
  site.spec.js                  ← all Playwright + axe tests
.github/workflows/
  test.yml                      ← new CI workflow (runs on every PR)
  lint.yml                      ← existing, unchanged
```

No changes to `index.html`, `styles.css`, or `script.js`.

Add to `.gitignore`:
```
node_modules/
playwright-report/
test-results/
```

## package.json

No `"type": "module"` — use CommonJS throughout. `devDependencies`: `@playwright/test`, `@axe-core/playwright`, `html-validate`, `serve`.

`scripts` block:
```json
{
  "test": "npm run test:html && npm run test:e2e",
  "test:html": "html-validate index.html",
  "test:e2e": "playwright test"
}
```

After creating `package.json`, run `npm install` locally to generate `package-lock.json`. Commit both files. CI uses `npm ci` which requires the lockfile.

## playwright.config.js

CommonJS syntax (no `"type": "module"` in `package.json`):

```js
const { defineConfig } = require('@playwright/test');

module.exports = defineConfig({
  testDir: './tests',
  use: { baseURL: 'http://localhost:3000' },
  webServer: {
    command: 'npx serve@14 . -l 3000',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
  },
  projects: [{ name: 'chromium', use: { browserName: 'chromium' } }],
  reporter: [['html', { open: 'never' }]],
});
```

`webServer` handles server startup and readiness — Playwright waits until the URL responds before running tests. Since `serve` is in `devDependencies`, `npx serve@14` resolves to the locally installed copy (no extra network fetch in CI).

`reuseExistingServer: !process.env.CI` is intentional: in CI (`process.env.CI=true`) Playwright always starts a fresh server. Locally, if port 3000 is already in use, Playwright reuses the existing server — this is standard Playwright behaviour and acceptable for local development.

## .htmlvalidate.json

```json
{
  "extends": ["html-validate:recommended"],
  "rules": {
    "no-style-tag": "off"
  }
}
```

Extends `html-validate:recommended`. `no-style-tag` is disabled because the `<noscript>` fallback intentionally contains a `<style>` element.

**Before committing `test.yml`:** run `npm run test:html` locally to confirm `index.html` passes with zero violations. If new failures appear beyond `no-style-tag`, add targeted rule overrides and document the reason in this file.

## Test Coverage

### E2E (Playwright — Chromium)

The site has four sections: `#intro`, `#services`, `#about`, `#contact`, all with matching nav links in `index.html`.

| Test | Assertion |
|---|---|
| Page title | `page.title()` contains substring `"Fredstam Consulting"` (`.toContain()`) |
| Meta description | `content` attribute of `meta[name="description"]` is non-empty |
| Nav links | Each of `a.nav-link[href="#intro"]`, `a.nav-link[href="#services"]`, `a.nav-link[href="#about"]`, `a.nav-link[href="#contact"]` has count `=== 1` |
| Skip link | `.skip-link` locator has count `>= 1` |
| Footer year | Evaluate `document.getElementById('footer-year').textContent` via `page.evaluate()` in the browser context and assert it equals `String(new Date().getFullYear())` — both evaluated in browser JS to use the same clock |
| External links | Every `a[target="_blank"]` has `rel` attribute containing both `"noopener"` and `"noreferrer"` |
| Hamburger open | **At 375×812 viewport:** clicking `#hamburger-btn` → `#site-nav` has class `is-open`, `#hamburger-btn` has `aria-expanded="true"` |
| Hamburger Escape | **At 375×812 viewport:** click `#hamburger-btn` to open, press `Escape` → `#site-nav` does not have class `is-open`, `#hamburger-btn` has focus |
| Hamburger nav link | **At 375×812 viewport:** click `#hamburger-btn` to open, click `page.locator('#site-nav .nav-link').first()` → `#site-nav` does not have class `is-open` |

All hamburger tests must set viewport to `375×812` before navigating, since the hamburger is only rendered at `max-width: 767px`.

### Accessibility (axe-core via `@axe-core/playwright`)

axe runs with `runOnly: { type: 'tag', values: ['wcag2a', 'wcag2aa'] }` — matching the WCAG 2.1 AA goal. `best-practice` rules are excluded.

| Test | Assertion |
|---|---|
| Desktop a11y | Full-page axe scan at 1280×720 — `violations` array is empty |
| Mobile a11y | Full-page axe scan at 375×812 — `violations` array is empty |

### HTML Validation (html-validate CLI)

`npm run test:html` runs `html-validate index.html` against `.htmlvalidate.json`. Any rule violation exits non-zero and fails the step.

## CI Workflow (`.github/workflows/test.yml`)

Triggers: `pull_request` (types: `opened`, `synchronize`, `ready_for_review`, `reopened`).

```yaml
permissions:
  contents: read
```

Steps:
1. `actions/checkout@v4`
2. `actions/setup-node@v4` with `node-version: '20'`
3. `npm ci`
4. `npx playwright install --with-deps chromium` (no manual caching — Chromium-only is fast enough)
5. `npm run test:html`
6. `npm run test:e2e` (Playwright starts/stops server via `webServer` config)
7. `actions/upload-artifact@v4` with `if: failure()`, path `playwright-report/`, `retention-days: 7`

`permissions: contents: read` is sufficient — `actions/upload-artifact` uses the GITHUB_TOKEN implicitly and does not require `actions: write` for artifact upload in GitHub's current default permissions model.

The existing `lint.yml` is unchanged. Lint and tests are independent PR status checks.

## Local Development

```bash
npm install         # first time: installs deps and generates package-lock.json
npm test            # full suite (html-validate + Playwright)
npm run test:html   # html-validate only (fast, no browser)
npm run test:e2e    # Playwright only (auto-starts server on localhost:3000)
```

## Constraints

- Zero changes to the served static files (`index.html`, `styles.css`, `script.js`, `fonts/`)
- `package.json`, `package-lock.json`, test files, and config files are dev infrastructure only — not deployed
- `npx` continues to work for the existing lint checks documented in `CLAUDE.md`
